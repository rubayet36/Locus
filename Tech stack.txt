Tech stack
React + Vite for the frontend, Supabase for everything backend (auth, database, realtime, edge functions, file storage).

Enable Row Level Security on every table so users only see papers in their group.

Supabase Edge Functions
You need three main edge functions (these call external APIs from the server side):
/lookup-paper — takes a DOI, URL, or title → calls Crossref API to get the ISSN and venue name → then calls Scimago (SJR) for journal rank, or CORE API for conference rank → returns a combined metadata object.
/check-access — takes a DOI → tries the publisher URL → if paywalled or 404, returns a Sci-Hub fallback link (https://sci-hub.se/{doi}).
/notify-assignment — sends an email via Supabase's built-in email when a paper is assigned to someone.

Frontend pages
Dashboard — activity feed showing who added/claimed/commented on what, recent papers.
Paper Search — a search bar accepting DOI, URL, or title. Shows a result card with rank badge (Q1 / A* etc.), abstract, authors, and buttons: Save to library, Assign to someone, Claim for reading, Open PDF.
Library — grid/table of all papers your group has saved. Filterable by rank, status, tags, who's reading it.
Reading Room — paper detail page with the PDF embedded (or Sci-Hub link), threaded comments on the side (realtime via Supabase subscriptions).
Assignments — Kanban-style board: Unread → Reading → Done. Shows who is reading what with a lock badge to prevent duplicates.

How the rank lookup flow works in practice

User pastes https://doi.org/10.1016/j.pragma.2011.10.004
Frontend strips the DOI → 10.1016/j.pragma.2011.10.004
Calls edge function /lookup-paper?doi=...
Edge function hits Crossref → gets ISSN 0378-2166, journal Journal of Pragmatics
Hits Scimago with ISSN → gets Q1, H-index 82
Caches result in paper_meta table (so the same paper isn't re-fetched)
Returns to frontend → shows a green Q1 badge
